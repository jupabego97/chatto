## Infrastructure

<RoadmapItem title="Zero-Dependency Deployment" status="done">
  The entire application — Go backend, embedded NATS server, and compiled
  SvelteKit frontend — ships as one executable. A NATS server runs embedded
  inside the process by default with JetStream for persistent storage, so
  there's nothing else to install or configure for a basic deployment. See the
  [Standalone Binary guide](/guides/binary/) for setup instructions.
</RoadmapItem>

<RoadmapItem title="Horizontal Scaling" status="done">
  Disable the embedded server and connect to an external NATS cluster instead.
  This enables multi-replica Chatto deployments behind a load balancer, all
  sharing the same message store. See the [Horizontal Scaling
  guide](/guides/horizontal-scaling/) for details.
</RoadmapItem>

<RoadmapItem title="S3 Asset Storage" status="done">
  Store file attachments in any S3-compatible object storage (AWS S3, Cloudflare
  R2, Backblaze B2, etc.) instead of NATS. Recommended for deployments with
  significant file upload traffic. See the [S3 Storage
  guide](/guides/s3-storage/) for setup instructions.
</RoadmapItem>

<RoadmapItem title="PWA Support" status="done">
  Chatto can be installed as a Progressive Web App on desktop and mobile. This
  provides an app-like experience with its own window, home screen icon, and
  support for push notifications.
</RoadmapItem>

<RoadmapItem title="High Availability" status="in-progress">
  Configure JetStream streams and KV buckets for replication across a multi-node
  NATS cluster. Server operators running a full NATS cluster will be able to
  ensure their data survives node failures without manual intervention. See the
  [High Availability guide](/guides/high-availability/) for current progress and
  configuration.
</RoadmapItem>

## Messaging

<RoadmapItem title="Real-time Chat" status="done">
  WebSocket-powered messaging with live updates across all connected clients.
  Messages appear instantly for everyone in the room without requiring a page
  refresh.
</RoadmapItem>

<RoadmapItem title="Threaded Conversations" status="done">
  Reply to any message to start an organized discussion thread. Threads open in
  a dedicated side pane and keep tangential conversations from cluttering the
  main room timeline.
</RoadmapItem>

<RoadmapItem title="Message Editing & Deletion" status="done">
  Edit your own messages within a configurable time window, or delete them
  entirely. Edits are clearly marked, and deletions use crypto-shredding — the
  encrypted content is destroyed, not just hidden, while a tombstone preserves
  conversation flow.
</RoadmapItem>

<RoadmapItem title="Emoji Reactions" status="done">
  React to messages with emoji, synchronized across all clients in real-time.
  Multiple users can add the same reaction, and counts are displayed inline on
  each message. An autocomplete dropdown lets you find emoji by name without
  leaving the keyboard.
</RoadmapItem>

<RoadmapItem title="@Mentions" status="done">
  Mention users with `@username` to get their attention. Mentioned users receive
  a notification and the mention is highlighted in the message body.
</RoadmapItem>

<RoadmapItem title="Typing Indicators" status="done">
  See when others are typing in a room or thread. Indicators appear in real-time
  and automatically expire so they never get stuck.
</RoadmapItem>

<RoadmapItem title="Direct Messages" status="done">
  Private one-on-one and group conversations with up to 10 participants. DM
  rooms are created on demand and persist across sessions so you never lose your
  conversation history.
</RoadmapItem>

<RoadmapItem title="File Attachments" status="in-progress">
  Upload images and files to any message. Images get automatic thumbnail
  generation with smart format selection (JPEG for photos, WebP for transparent
  images, GIF for animations). Video uploads and animated GIF support are in
  progress. See the [Video Processing guide](/guides/video-processing/) for
  enabling video transcoding.
</RoadmapItem>

<RoadmapItem title="Mention Autocomplete" status="in-progress">
  Type `@` in the message composer to get a filtered list of users in the
  current room, making it easy to mention someone without remembering their
  exact username.
</RoadmapItem>

<RoadmapItem title="Link Previews & Media Embeds" status="in-progress">
  Automatically fetch and display rich previews for URLs shared in messages —
  article titles, descriptions, and thumbnail images. Media links (YouTube,
  etc.) embed inline so users can preview content without leaving the
  conversation.
</RoadmapItem>

<RoadmapItem title="Message Permalinks" status="planned">
  Generate a direct, shareable link to any specific message. Clicking a
  permalink scrolls the room to that message and highlights it for context.
</RoadmapItem>

<RoadmapItem title="Custom Emoji" status="planned">
  Server admins can upload custom emoji that are available to all members.
  Custom emoji work everywhere regular emoji do — in messages, reactions, and
  autocomplete.
</RoadmapItem>

<RoadmapItem title="GIF Chooser" status="planned">
  A built-in GIF picker in the message composer powered by Tenor or Giphy.
  Browse trending GIFs or search by keyword and insert them into messages with
  one click.
</RoadmapItem>

## Servers & Rooms

<RoadmapItem title="Servers & Rooms" status="done">
  Each Chatto deployment is a single server — a community with its own member
  list, roles, permissions, and branding (custom logo and banner). Servers
  contain rooms (channels and direct messages) where conversations happen.
</RoadmapItem>

<RoadmapItem title="Unread & Mention Indicators" status="done">
  Visual indicators in the sidebar show which rooms have unread messages.
  Direct @mentions get a distinct highlight so you can prioritize conversations
  that need your attention.
</RoadmapItem>

<RoadmapItem title="Room Reordering, Grouping & Archiving" status="done">
  Drag-and-drop room list customization so you can put your most-used rooms at
  the top. Group rooms into categories and archive inactive ones to keep the
  sidebar focused. Room order and groups are saved per-user so everyone can
  organize their sidebar how they like.
</RoadmapItem>

## Notifications

<RoadmapItem title="In-App Notifications" status="done">
  A built-in notification center collects @mentions, thread replies, and DM
  messages in one place. Notifications link directly to the relevant message so
  you can jump into context with one click.
</RoadmapItem>

<RoadmapItem title="Web Push Notifications" status="done">
  Receive browser push notifications when the app is in the background or
  closed. Push notifications work on desktop and mobile browsers that support
  the Web Push API.
</RoadmapItem>

<RoadmapItem title="Notification Preferences & Muting" status="done">
  Granular control over which events trigger notifications — globally or
  per-room. Mute specific rooms to stop receiving notifications from them
  entirely. Read state and notification preferences sync across all your
  devices in real-time.
</RoadmapItem>

## Users & Authentication

<RoadmapItem title="Authentication" status="done">
  Self-service account registration, email verification, and password reset.
  Users can also sign in via OIDC providers (e.g. Chatto Hub).
  Server admins can configure whether email is required and whether
  verification is enforced.
</RoadmapItem>

<RoadmapItem title="User Profiles & Presence" status="done">
  Each user has a profile with a display name, custom avatar, and presence
  status (Online, Away, Do Not Disturb). Presence automatically expires after
  inactivity so stale statuses don't linger. Profiles are visible to all
  authenticated users on the server.
</RoadmapItem>

## Authorization & Permissions

<RoadmapItem title="Role-Based Access Control" status="done">
  A permission system spanning the server and individual rooms. Custom roles
  carry fine-grained permissions and a position-based hierarchy where
  higher-ranked roles take precedence. Room-level overrides enable patterns
  like read-only announcement channels or restricted rooms with different
  moderation rules.
</RoadmapItem>

<RoadmapItem title="Role & Group Pinging" status="planned">
  Use `@role` syntax to mention and notify all users with a specific role.
  Useful for pinging all moderators, a project team, or any other defined group
  without listing individuals.
</RoadmapItem>

## Administration

<RoadmapItem title="Server Admin Panel" status="done">
  A dedicated admin interface for managing the server. View and manage users,
  rooms, roles, and permissions from a single dashboard. Configure server name
  and description, message of the day, welcome message, and blocked usernames —
  changes take effect immediately without restarting the process. Includes NATS
  monitoring for JetStream streams, KV buckets, and connection metrics.
</RoadmapItem>

<RoadmapItem title="Backup & Restore" status="done">
  CLI commands for backing up and restoring all server data. Backups
  intentionally exclude encryption keys so that exported data remains encrypted
  at rest. See the [Backup & Restore guide](/guides/backup-restore/) for
  details.
</RoadmapItem>

<RoadmapItem title="Full-Text Search" status="planned">
  Search across message history to find past conversations, decisions, and
  shared links. Results are scoped by the user's permissions so you only see
  messages from rooms you have access to.
</RoadmapItem>

<RoadmapItem title="Message Retention Policies" status="planned">
  TTL-based automatic message cleanup. Configure how long messages are retained
  per room or globally, and let the system automatically purge expired content
  to manage storage costs.
</RoadmapItem>

## Security & Encryption

<RoadmapItem title="Per-User Encryption" status="done">
  Every message body is encrypted at rest with XChaCha20-Poly1305 using the
  author's active message-body DEK epoch, and durable user PII events use a
  separate user-PII DEK epoch. DEKs are wrapped behind Chatto's KMS
  boundary, stored by the application under opaque content-key refs, and only unwrapped when
  authorized projections or API paths need plaintext. Shredding a user's
  content-key refs or wrapping-key refs renders their encrypted data permanently unreadable — fast, reliable
  GDPR-compliant data destruction without needing to locate individual messages
  across streams.
</RoadmapItem>

<RoadmapItem title="Signed Asset URLs" status="done">
  All file attachment URLs are cryptographically signed with a server-side
  secret. This prevents URL tampering and ensures that only the server can
  generate valid download links.
</RoadmapItem>

<RoadmapItem title="Key Management" status="planned">
  Run the key management service as a standalone process for high-security
  deployments, with key material isolated on dedicated infrastructure or backed
  by hardware security modules. Includes key export and escrow for disaster
  recovery, so a lost server doesn't mean permanent data loss.
</RoadmapItem>

## Voice & Video

<RoadmapItem title="Voice & Video Calls" status="in-progress">
  Real-time voice and video calls inside rooms, powered by LiveKit as a
  Selective Forwarding Unit (SFU). Small groups connect with low latency, and
  the SFU architecture scales to larger meetings without requiring peer-to-peer
  connections between every participant. See the [Voice & Video Calls
  guide](/guides/voice-calls/) for setup instructions.
</RoadmapItem>

<RoadmapItem title="Screen Sharing" status="planned">
  Share your screen or a specific application window during a call. Other
  participants see the shared content as an additional video stream alongside
  the speaker's camera feed.
</RoadmapItem>

<RoadmapItem title="Room Huddles" status="in-progress">
  Lightweight, always-on voice channels that room members can drop in and out of
  at any time — similar to Slack huddles or Discord voice channels. No need to
  initiate a call; just join when you want to talk and leave when you're done.
</RoadmapItem>

## APIs & Extensibility

<RoadmapItem title="GraphQL API" status="done">
  A full-featured GraphQL API with queries, mutations, and subscriptions for
  building client applications and bot integrations. Supports real-time updates
  over WebSocket and covers all user-facing functionality including messaging,
  rooms, and permissions.
</RoadmapItem>

<RoadmapItem title="NATS API" status="planned">
  A NATS-based API for extending Chatto with custom services and integrations.
  Connect directly to the NATS message bus to subscribe to events, publish
  messages, and interact with JetStream streams — ideal for building backends
  that need tight, low-latency integration with the system.
</RoadmapItem>

<RoadmapItem title="Bot Accounts" status="planned">
  Dedicated bot user accounts that can be created and managed by server
  admins. Bots authenticate via API tokens instead of passwords and can post
  messages, react to events, and participate in rooms just like regular users.
</RoadmapItem>

<RoadmapItem title="Programmable UI" status="planned">
  Allow bots to render custom interactive UI elements — buttons, forms, cards —
  directly inside chat messages. Users can interact with these elements without
  leaving the conversation, enabling workflows like approvals, polls, and ticket
  management right in the chat.
</RoadmapItem>

## Not Planned

These are features we've considered but decided against — either because they don't fit the project's scope, or because the complexity isn't worth the value they'd add.

<RoadmapItem title="Game Streaming" status="not-planned">
  Built-in game streaming or screen broadcasting to large audiences. Chatto is a
  chat application, not a streaming platform — dedicated tools like OBS, Twitch,
  or Discord's Go Live do this much better. Screen sharing during calls covers
  the collaboration use case.
</RoadmapItem>

<RoadmapItem title="End-to-End Encryption" status="not-planned">
  Client-side encryption where the server never sees plaintext. Chatto uses
  server-side per-user encryption which provides strong privacy guarantees
  (including crypto-shredding) while keeping features like search, moderation,
  and notifications possible. True E2EE would break most of these and add
  significant complexity for marginal benefit in a self-hosted context where you
  already control the server.
</RoadmapItem>

<RoadmapItem title="Federation" status="not-planned">
  Connecting multiple Chatto servers into a federated network (like Matrix or
  ActivityPub). Federation adds enormous protocol complexity and makes it much
  harder to guarantee message delivery, encryption, and permission semantics.
  Chatto focuses on being excellent for a single organization or community.
</RoadmapItem>
