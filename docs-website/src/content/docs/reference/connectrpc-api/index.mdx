Chatto exposes these ConnectRPC services below `/api/connect`.

Endpoint paths use the Connect convention:

`/api/connect/<fully-qualified-service>/<method>`

Use this reference when building clients against Chatto's public ConnectRPC API.

## Services

### [NotificationPreferencesService](notification-preferences-service/)

Reads and updates per-room notification preferences for the current user.

| Method | Description |
| --- | --- |
| [`GetRoomNotificationPreference`](notification-preferences-service/#getroomnotificationpreference) | Returns the current user's explicit and effective notification level for a room. Use this before rendering room notification controls so the UI can distinguish inherited defaults from an explicit room override. |
| [`SetRoomNotificationLevel`](notification-preferences-service/#setroomnotificationlevel) | Sets the current user's explicit notification level for a room and returns the resulting stored and effective levels. |


### [ReadStateService](read-state-service/)

Updates room and thread read state for the current user.

| Method | Description |
| --- | --- |
| [`MarkRoomAsRead`](read-state-service/#markroomasread) | Marks a room timeline as read through the supplied event. Clients usually call this after the user has viewed the latest visible event in the room. |
| [`MarkThreadAsRead`](read-state-service/#markthreadasread) | Marks a thread timeline as read through the supplied event without changing the room-level read marker. |


### [RoomTimelineService](room-timeline-service/)

Reads room and thread timeline pages.

| Method | Description |
| --- | --- |
| [`GetRoomEvents`](room-timeline-service/#getroomevents) | Returns one page of room timeline events, including related user data needed to render the page. |
| [`GetRoomEventsAround`](room-timeline-service/#getroomeventsaround) | Returns a room timeline window centered around a specific event. Use this to open a permalink, search result, or notification target in context. |
| [`GetThreadEvents`](room-timeline-service/#getthreadevents) | Returns one page of events in a message thread. Initial pages include the thread root message; cursor pages return replies in the requested direction. |
| [`GetThreadEventsAround`](room-timeline-service/#getthreadeventsaround) | Returns a thread timeline window centered around a specific event. Use this to open a reply from a notification or search result in context. |


### [ServerService](server-service/)

Provides public server discovery metadata.

| Method | Description |
| --- | --- |
| [`GetServer`](server-service/#getserver) | Returns public server metadata, branding, registration status, and login provider information. This RPC is available before login so clients can render the first server screen and authentication choices. |


### [ThreadService](thread-service/)

Manages thread follow state for the current user.

| Method | Description |
| --- | --- |
| [`FollowThread`](thread-service/#followthread) | Follows a thread for the current user. Followed threads can be surfaced in clients and can participate in thread notification behavior. |
| [`UnfollowThread`](thread-service/#unfollowthread) | Stops following a thread for the current user. The response reports the resulting follow state so clients can update local UI immediately. |


### [UserStatusService](user-status-service/)

Manages the current user's custom profile status.

| Method | Description |
| --- | --- |
| [`SetCustomStatus`](user-status-service/#setcustomstatus) | Sets or replaces the current user's custom status. Emoji and text are required, and expires_at must be omitted or in the future. |
| [`ClearCustomStatus`](user-status-service/#clearcustomstatus) | Clears the current user's custom status. The call is idempotent and returns the resulting empty status state. |
